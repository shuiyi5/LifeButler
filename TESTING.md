# 测试策略

## 1. 测试方法：TDD（测试驱动开发）
每个功能开发流程：
1. 先写测试 → 运行确认失败（红灯 🔴）
2. 写最少代码让测试通过（绿灯 🟢）
3. 重构优化（保持绿灯 🟢）

## 2. 测试框架
| 类型 | 框架 | 配置 |
|------|------|------|
| 后端单元测试 | pytest + pytest-asyncio | `pytest.ini` / `pyproject.toml` |
| 后端集成测试 | pytest + httpx (AsyncClient) | 使用测试数据库 |
| 前端单元测试 | Jest + React Testing Library | `jest.config.ts` |
| 前端 E2E 测试 | Playwright | `playwright.config.ts` |
| API 测试 | pytest + httpx | 测试 FastAPI 路由 |

### 测试数据库
- 使用独立的 PostgreSQL 测试数据库 `lifebutler_test`
- 每个测试用例前自动清空数据
- 使用 factory 模式生成测试数据
