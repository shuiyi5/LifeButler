# 需求规格说明书

## 1. 项目背景

LifeButler（生活管家）是一个基于 AI 的个人生活管理平台，旨在通过自然语言对话的方式，帮助用户高效管理日常生活的方方面面。与传统的工具类应用不同，LifeButler 采用「主动关怀」模式，会在用户登录时智能汇总待处理事项并主动推送，如同一位贴心的真人管家。

## 2. 目标用户
- **用户画像**：个人用户，注重生活品质，希望用一个统一入口管理所有生活事务
- **使用场景**：日常碎片化使用（随时记账、查天气、记待办）+ 固定时间使用（早晨看日报、晚上复盘总结）
- **技术水平**：普通用户，无技术背景，期望通过自然语言完成所有操作
- **用户规模**：初期个人使用，未来扩展到几十人（需预留多账户架构）

## 3. 核心功能需求

### F-001: 自然语言对话
- **描述**：用户通过自然语言与管家交互，管家自动识别意图并调用对应功能
- **优先级**：P0
- **输入**：用户文字消息
- **输出**：管家文字回复 + 执行对应操作
- **验收标准**：
  - [ ] 支持流式响应输出
  - [ ] 能正确识别 15 个功能模块的意图
  - [ ] 对话支持上下文连续性（记住最近的对话历史）
  - [ ] 无法识别时友好提示引导

### F-002: 日程管理
- **描述**：创建、查询、修改、删除日程事件，支持重复事件和冲突检测
- **优先级**：P0
- **输入**：自然语言描述（如"明天下午3点开会"）
- **输出**：日程创建/查询结果
- **验收标准**：
  - [ ] 支持创建单次和重复事件
  - [ ] 支持按日/周/月查询日程
  - [ ] 时间冲突时主动提醒
  - [ ] 支持修改和删除已有日程

### F-003: 待办管理
- **描述**：任务的增删改查，支持优先级、截止日期、分类标签
- **优先级**：P0
- **输入**：自然语言指令（如"添加一个待办：买菜，优先级高"）
- **输出**：任务操作结果 + 待办列表
- **验收标准**：
  - [ ] 支持增删改查
  - [ ] 支持优先级（高/中/低）
  - [ ] 支持截止日期和过期提醒
  - [ ] 支持按状态/优先级筛选

### F-004: 智能提醒
- **描述**：用户登录时主动汇总推送待处理事项，包括天气变化、待办截止、纪念日、账单等
- **优先级**：P0
- **输入**：用户登录事件
- **输出**：智能汇总卡片
- **验收标准**：
  - [ ] 登录时自动生成"今日概览"
  - [ ] 包含天气、待办、日程、提醒等多维度信息
  - [ ] 内容根据时间段动态调整（早/中/晚不同内容）
  - [ ] 支持用户自定义关注项

### F-005: 天气查询
- **描述**：实时天气查询、未来天气预报、穿衣建议
- **优先级**：P1
- **输入**：城市名或"今天天气"
- **输出**：天气信息 + 生活建议
- **验收标准**：
  - [ ] 支持实时天气和 7 日预报
  - [ ] 提供穿衣、出行建议
  - [ ] 极端天气主动提醒
  - [ ] 支持设置默认城市

### F-006: 新闻摘要
- **描述**：每日热点新闻聚合与摘要
- **优先级**：P2
- **输入**："今日新闻" 或特定领域查询
- **输出**：新闻摘要列表
- **验收标准**：
  - [ ] 支持多新闻源聚合
  - [ ] AI 自动生成摘要
  - [ ] 支持按分类筛选

### F-007: 记账理财
- **描述**：收支记录、分类统计、月度报表、预算管理
- **优先级**：P0
- **输入**：自然语言（如"午饭花了35块"）
- **输出**：记账确认 + 统计数据
- **验收标准**：
  - [ ] 自然语言自动解析金额、分类、日期
  - [ ] 支持收入和支出记录
  - [ ] 提供日/周/月统计和图表
  - [ ] 支持设置月度预算和超支预警

### F-008: 饮食健康
- **描述**：饮食记录、营养分析、饮水提醒
- **优先级**：P1
- **输入**：自然语言（如"中午吃了一碗牛肉面"）
- **输出**：饮食记录确认 + 营养分析
- **验收标准**：
  - [ ] 支持记录三餐和零食
  - [ ] 提供简单的热量/营养估算
  - [ ] 登录时提供饮水提醒
  - [ ] 周度/月度饮食报告

### F-009: 习惯打卡
- **描述**：创建习惯追踪、每日打卡、连续统计
- **优先级**：P1
- **输入**：自然语言（如"创建习惯：每天跑步" 或 "跑步打卡"）
- **输出**：打卡确认 + 统计数据
- **验收标准**：
  - [ ] 支持创建/删除自定义习惯
  - [ ] 支持每日打卡
  - [ ] 显示连续打卡天数和完成率
  - [ ] 断签提醒

### F-010: 购物清单
- **描述**：商品添加、分类管理、购买勾选
- **优先级**：P1
- **输入**：自然语言（如"购物清单加上牛奶和面包"）
- **输出**：清单更新确认
- **验收标准**：
  - [ ] 支持按分类（食品/日用/电器等）自动归类
  - [ ] 支持勾选已购买
  - [ ] 支持清空已购买项
  - [ ] 显示清单概览

### F-011: 读书学习
- **描述**：读书计划创建、进度追踪、学习提醒
- **优先级**：P2
- **输入**：自然语言（如"开始读《原则》，一共300页，计划两周读完"）
- **输出**：计划创建/进度更新确认
- **验收标准**：
  - [ ] 支持创建读书/学习计划
  - [ ] 支持记录每日进度
  - [ ] 进度落后时主动提醒
  - [ ] 完成时祝贺通知

### F-012: 智能家居控制
- **描述**：通用智能家居设备控制接口（预留架构）
- **优先级**：P2
- **输入**：自然语言（如"开灯"、"空调调到26度"）
- **输出**：操作确认
- **验收标准**：
  - [ ] 设计通用设备适配器接口
  - [ ] 提供 Mock 设备用于测试
  - [ ] 支持未来接入具体平台（米家/HomeKit 等）
  - [ ] 自然语言正确解析设备和操作

### F-013: 邮件管理
- **描述**：QQ 邮箱收件摘要、智能分类、快捷回复
- **优先级**：P1
- **输入**："查看邮件" 或自然语言回复指令
- **输出**：邮件摘要列表 + 回复确认
- **验收标准**：
  - [ ] 通过 IMAP 读取 QQ 邮箱未读邮件
  - [ ] AI 自动生成邮件摘要
  - [ ] 支持快捷回复（AI 起草，用户确认后发送）
  - [ ] 支持 SMTP 发送邮件

### F-014: 出行规划
- **描述**：路线推荐、实时交通、周边搜索
- **优先级**：P1
- **输入**：自然语言（如"从家到公司怎么走最快"）
- **输出**：路线方案 + 预估时间
- **验收标准**：
  - [ ] 支持驾车/公交/步行多种方案
  - [ ] 显示实时交通状况和预计用时
  - [ ] 支持 POI 周边搜索（如"附近的咖啡店"）
  - [ ] 支持设置常用地址（家、公司）

### F-015: 备忘笔记
- **描述**：快速记录想法，自动同步到 Notion
- **优先级**：P1
- **输入**：自然语言（如"记一下：项目方案要改需求文档"）
- **输出**：笔记保存确认 + Notion 同步状态
- **验收标准**：
  - [ ] 支持快速创建笔记
  - [ ] 支持按日期/标签查询历史笔记
  - [ ] 自动同步到 Notion 指定数据库
  - [ ] Notion 同步失败时本地保留不丢失

## 4. 非功能性需求

### 4.1 性能
- 响应时间：无硬性要求，AI 回复需有流式输出避免等待感
- 并发量：初期个人使用，架构支持未来几十人同时在线

### 4.2 安全
- 用户认证：邮箱 + 密码注册登录，JWT Token 鉴权
- 数据加密：敏感数据（财务、邮件、笔记）AES-256 加密存储
- API Key 保护：所有第三方 API Key 通过环境变量管理，不入代码库
- 密码安全：bcrypt 哈希存储，不可逆

### 4.3 可用性
- 温暖治愈的 UI 风格，降低使用门槛
- 自然语言交互，零学习成本
- 响应式设计，支持手机浏览器访问

## 5. 约束条件
- 月度运营预算 ≤ ¥100（服务器 + API 调用）
- 开发时间目标：一周内完成
- Web 端优先，移动端和桌面端后续迭代
- 部署环境：腾讯云轻量服务器 + Docker

## 6. 技术边界
- **能做**：自然语言管理日程/财务/健康/出行等 15 大生活场景，主动提醒，Notion 同步，邮件收发，天气/新闻聚合
- **不能做**：实时语音对话（仅文字）、支付/转账（仅记录）、实际打车下单（仅路线建议）、实际智能家居控制（仅预留接口，需接入实际设备）
